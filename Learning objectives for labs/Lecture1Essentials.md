### Fundamental Topics First‑Year Engineering Students Should Master — Lecture 1

*Difficulty ratings: 1 = easiest, 3 = most challenging*

1. Translating Problems Into MATLAB Workflows — **Difficulty: 2**
   - Break an engineering problem into steps: define variables, perform computations, and present results programmatically (scripts or command‑line sequence).
   - *Requires problem‑solving and planning, but uses basic MATLAB commands.*

2. Command Window and Interpreter Basics — **Difficulty: 1**
   - Enter and evaluate expressions interactively, read error messages, use semicolons to suppress output, and inspect immediate results.
   - *Low conceptual load; hands‑on practice suffices.*

3. Command History and Rapid Iteration — **Difficulty: 1**
   - Reuse and edit previous commands with the up‑arrow and Command History to accelerate exploratory work and debugging.
   - *Tool‑usage skill; easy to learn by practice.*

4. Variable Creation, Assignment, and Workspace Inspection — **Difficulty: 1**
   - Create variables with valid identifiers, assign numeric/text values, and inspect workspace with `who`/`whos` and `clear`/`clearvars` to manage state.
   - *Basic syntax and workspace commands; minimal difficulty.*

5. Identifier Rules and Naming Best Practices — **Difficulty: 1**
   - Use valid, descriptive names (no leading digits or spaces), avoid shadowing built‑ins, and follow consistent naming conventions for readability.
   - *Simple rules and conventions; primarily style guidance.*

6. Numeric Types, Literals, and Expression Evaluation — **Difficulty: 2**
   - Work with default numeric class (double), form expressions with correct operator precedence, and understand type conversion basics.
   - *Some subtleties (type conversion, precedence) require attention.*

7. Common Constants and Built‑In Math Functions — **Difficulty: 1**
   - Use constants (pi, eps) and functions like `sin`, `cos`, `sqrt`, `exp`, `abs`, `round` correctly in computations and know their input/output behavior.
   - *API familiarity and straightforward usage.*

8. Basic Number Representation Concepts — **Difficulty: 3**
   - Understand binary/finite‑precision basics and practical implications for rounding and numerical accuracy (conceptual + simple use of `dec2bin`/`bin2dec`).
   - *Concepts of binary representation and finite precision are abstract and can be counterintuitive.*

9. Text Basics: String Scalars and Character Vectors — **Difficulty: 2**
   - Create and distinguish string (`"..."`) and char (`'...'`) types, test with `isstring`/`ischar`, and perform simple concatenation and display.
   - *Two similar types and indexing differences add modest complexity.*

10. Scripts, Comments, and Reproducible Code Style — **Difficulty: 2**
    - Write readable scripts with clear comments, sensible structure, avoid relying on persistent workspace state, and use `clc`/`clear` judiciously.
    - *Requires discipline and good practices; slightly higher cognitive load than syntax alone.*

11. Simple Functions and Basic User Input — **Difficulty: 2**
    - Create function files with inputs/outputs, call them from scripts/command line, accept validated user input with `input`, and handle wrong input robustly.
    - *Involves file structure, I/O, and input validation—conceptual but core skills.*
